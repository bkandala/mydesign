# Contributing to mydesign

Thank you for contributing to the design system.

## Ways to contribute
- Report bugs
- Suggest enhancements
- Improve documentation
- Add or refine design tokens
- Build and improve reusable UI components

## Development workflow
1. Create a branch from the default branch.
2. Make focused changes.
3. Update docs when behavior or APIs change.
4. Open a pull request with a clear summary.

## Pull request checklist
- [ ] Code is documented where needed
- [ ] Documentation was updated
- [ ] Changes are scoped and easy to review
- [ ] Visual or API changes are described clearly

## Commit guidance
Use clear, descriptive commit messages.
